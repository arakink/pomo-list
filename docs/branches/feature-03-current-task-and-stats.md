# `feature/03-current-task-and-stats`

## 目的

- タイマー下に現在のタスク表示を追加する
- タイマー下にタグ別の完了回数表示を追加する
- ToDo 連携前の段階として、表示の土台を先に整える

## このブランチで含めるもの

- `Current Task` セクションの追加
- `Tag Stats` セクションの追加
- 仮データまたは空状態で成立する UI
- 状態ごとに破綻しない文言とレイアウトの整理

## 状態別 UI 方針

### `Current Task`

- タスクあり・タグあり
  - タスク名を表示する
  - タグを通常表示する
- タスクなし
  - セクション自体は表示したままにする
  - タスク表示位置に `未設定` を表示する
  - 空状態であることが自然に伝わる補足文言を置く
- タスクあり・タグなし
  - タスク名は表示する
  - タグの表示位置は残し、`タグなし` を表示する

### `Tag Stats`

- 統計データあり
  - タグ別の完了回数を一覧表示する
- 統計データなし
  - セクション自体は表示したままにする
  - まだ集計がないことが分かる空状態を表示する
- タグ未設定タスクの完了回数
  - `タグなし` として集計対象に含める

## 操作方針

- このブランチでは `Current Task` は表示専用とする
- タスク名やタグの追加・編集は `TodoPanel` 側で行う
- `Current Task` セクションから直接編集や追加は行わない

## データの考え方

- この段階では仮データまたは空状態でよい
- 表示の土台を先に作り、実際の ToDo 連携は次ブランチで追加する
- 永続化は `feature/05-polish-persistence` まで行わない

## このブランチでやらないこと

- ToDo から現在のタスクをセットする操作
- タイマー完了時にタスクやタグへ回数を加算する処理
- `Current Task` から直接タスクを編集する操作
- `localStorage` 保存
- 通知や効果音

## 完了条件

- タイマー下に `Current Task` セクションが表示される
- タイマー下に `Tag Stats` セクションが表示される
- `Current Task` がタスクあり・なしの両方で自然に見える
- タグがない場合でもレイアウトが崩れない
- `Tag Stats` がデータあり・なしの両方で自然に見える
- ToDo 連携なしでも単体で動作する
