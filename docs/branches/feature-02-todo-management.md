# `feature/02-todo-management`

## 目的

- ToDo を自分で追加できる
- タスクを未完了 / 完了で分けて管理できる
- 各タスクにタグを持たせられる
- 一覧を見れば今のタスク状況が分かる

## このブランチで含めるもの

- ToDo の新規追加
- タスク名の入力
- タグの入力
- 未完了タスク一覧の表示
- 完了タスク一覧の表示
- タスク名とタグの編集
- タスクの完了切替
- 完了済みタスクを未完了へ戻す操作
- タスクの削除

## UI の想定

- 入力エリア
- `タスク名`
- `タグ`
- `追加` ボタン
- 一覧エリア
- `未完了`
- `完了済み`
- 各 ToDo に状態変更ボタンを置く
- 各 ToDo に編集と削除の操作を置く

## データの考え方

- 1件の ToDo は `id` `title` `tag` `completed` を持つ
- この段階ではメモリ上の state 管理でよい
- 永続化は `feature/05-polish-persistence` で対応する

## このブランチでやらないこと

- タイマーへのタスクセット
- active task 表示
- タグ別の統計集計
- `localStorage` 保存
- 通知や効果音

## 完了条件

- タスクを追加すると未完了一覧に表示される
- 既存タスクのタスク名とタグを更新できる
- 完了操作で完了済み一覧へ移動する
- 戻す操作で未完了一覧へ戻せる
- 不要なタスクを削除できる
- タグ付きタスクが一覧上で識別できる
- タイマー機能に依存せず単体で動作する
