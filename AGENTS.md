# AGENTS.md

## プロジェクト概要

PomoList は ToDo 管理を備えたポモドーロタイマーアプリです。
プロダクトの方向性と実装順は `docs/implementation-plan.md` にまとめています。

## 重要なドキュメント

- ブランチ順と実装フェーズは `docs/implementation-plan.md` を参照する
- ブランチごとの詳細は `docs/branches/` を参照する

## 作業ルール

- 変更は現在の feature ブランチの責務に収める
- 大きなリファクタより、小さく段階的な変更を優先する
- 明示的な依頼がない限り、既存の見た目の方向性は保つ
- 現在のブランチでより広い状態管理が必要になるまでは、まず単純なローカル state を優先する
- `feature/05-polish-persistence` より前に永続化を追加しない
- `feature/04-timer-todo-integration` より前にタイマーと ToDo の連携を追加しない

## 実装時の方針

- 新しい実装は `docs/implementation-plan.md` にある各ブランチの責務に合わせる
- UI や state を追加するときは、そのブランチ固有のスコープを明確に保つ
- 要件が曖昧な場合は、現在のブランチに収まるより単純な実装を優先する

## 確認

- コード変更後は、可能な範囲で関連する確認を行う
- このプロジェクトでは、基本の確認コマンドとして `npm run lint` を優先する
